Fox Alien Masuter + Candle (GRBL)
=================================

Router: GRBL controller, USB serial (often CH341/CH340 — shows as USB-SERIAL CH340 in Device Manager).

1) USB driver (once)
   If Windows does not assign a COM port, install the vendor driver from WCH:
   https://www.wch-ic.com/downloads/CH341SER_EXE.html
   Replug USB and confirm a COM port appears under Ports (COM & LPT).

2) Start Candle
   Double-click Run-Candle-Masuter.bat (or run candle-11.2\Candle\candle.exe).

3) Connect in Candle
   Service > Settings (or connection panel):
   - Port: COM from Device Manager (e.g. COM7). Your CH340 assignment can change after replug/USB hub.

   - Baud rate: 115200 (standard for GRBL 1.1)
   Apply and connect. Status should show Grbl and position updates.

4) Machine / workspace (first run)
   Set soft limits and travel to match your Masuter model in the firmware/Candle as needed.
   Homing: only if you have limit switches wired — use controls carefully.

5) Sending jobs
   File > Open to load NC/G-code, then use the send/run controls after checking origin and Z clearance.

Reinstall Candle: run install-candle.ps1 (downloads v11.2 portable from GitHub).

Official Candle project: https://github.com/Denvi/Candle

6) Nothing moves — COM stuck / Idle confusion
   - Idle (or <Idle …> from `?`) is GOOD: the controller is ready. You do NOT need to clear Idle.
   - If Candle is connected first, Disconnect-Candle.ps1 (or disconnect in Candle UI) frees CH340 COM.
   - Orphan PowerShell tests can also hang COM until closed; unplug/replug USB as a last resort.
   - Scripts: Disconnect-Candle.ps1 then DryRun-WebCircle.ps1 streams every G/M line; **whatever is in your file runs** (`M5` spindle off versus `M3` spindle ON).
7) PowerShell one-liners: avoid naming helpers `R` (and similar)
   PowerShell binds `R` to the `Invoke-History` alias; calls like `(R $port 5000)` can error with "positional parameter … Invoke-History". Use spelled-out names (`ReadFor`, `Send-Line`, etc.).

8) Shallow smiley carve into soft pine (`samples/smiley-face-soft-pine-G54.mm.nc`)
   Uses G54: Z0 MUST be TOP of stock; XY origin at board centre beneath the spindle.
   Typical flow: `$H`; Run-Smiley-Carve.ps1 -OnlyCenterTable (disconnect Candle) for machine table-centre XY; jog bit over VISUAL pine centre then zero XY in Candle;
   jog Z gently to touch TOP of pine thin-paper-drag then zero Z; CLOSE Candle; Run-Smiley-Carve.ps1 then type CUT to stream spindle-on job.
   Edit sizes/feeds via New-SmileyFacePineNc.ps1 (-DepthMm, -SpinupDwellSeconds, -StepDownMm, etc.).
   NC order (**important**): first commands are **`M3 S…`** **then **`G4 P…` dwell`** **then **`G21`…`G54` setup`**, **then **`G0 Z` Safe`** before carve XY tiers.
   Multi-pass carve uses step tiers to lighten chip/snag loads. If `M3` streams but spindle never electrically runs, troubleshoot GRBL spindle/PWM routing vs Cartesian motion wiring.
